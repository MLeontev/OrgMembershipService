# Процессы и границы проекта

## Процессы

1. **Управление организациями:** Бизнес-процесс отвечает за подключение организации к системе.

   В рамках этого направления реализуется создание и конфигурация организации, изоляция данных между разными организациями.

   Создание организации: юзеры смогут регистрироваться и без организации. А вот при создании надо будет указать конкретного пользователя как ответственного за организацию.

   Создать организацию сможет любой пользователь. Сюда же входит управление жизненным циклом организации: создание, удаление, …

2. **Управление пользователями и ролями:** Бизнес-процесс отвечает за работу с сотрудниками внутри организации.

   В него входит добавление и приглашение пользователей, назначение ролей и прав доступа.

3. **Управление ресурсами:** Бизнес-процесс описывает и поддерживает корпоративные ресурсы, доступные для бронирования.

   В рамках направления реализуется создание, редактирование и поддержка актуальной информации о ресурсах (адрес офиса, этаж, переговорные комнаты, рабочие места, оборудование).

   Также здесь есть управление статусами ресурсов (Например: доступна, выведена из работы, временно недоступно). Особые правила бронирования для некоторых ресурсов.

4. **Бронирование и использование ресурсов:** Бизнес-процесс отвечает за основной пользовательский сценарий -
   бронирование ресурсов.

   В него входит поиск доступных ресурсов, создание и отмена бронирований, проверка конфликтов по времени.

   Процесс бронирования выполняется с учётом политик бронирования, представляющих собой набор бизнес-правил, определяющих условия использования ресурсов.

   Политики могут учитывать ограничения по времени, лимиты бронирований для пользователей и приоритеты ролей.

   В случае конфликтов или нарушений правил система автоматически принимает решение об отклонении запроса

## Границы проекта

В рамках проекта рассматривается сценарий внутреннего бронирования ресурсов внутри организации.

Мы не реализуем сложные процессы согласования с участием нескольких ролей, интеграции с внешними системами, а также дублирование заявок во внешние контуры.

Проект не предполагает учёт финансовых, бухгалтерских или эксплуатационных аспектов ресурсов и ориентирован на базовый, понятный пользовательский сценарий бронирования.
