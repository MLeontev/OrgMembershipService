# OrgMembershipService

## Контекст
Описание контекста системы в целом представлено [здесь](SystemContext.md)

OrgMembershipService - Сервис управления пользователями и ролями в организации

Сервис отвечает за:
- пользователей (глобальный аккаунт)
- членство в организациях
- роли и права (RBAC)
- приглашения (invite-link)

### Роли
- Есть системные роли (OrganizationId = NULL)
- В организациях можно создавать кастомные роли (OrganizationId = orgId)
- У участника в org может быть несколько ролей одновременно

### Организации и членство
- Пользователь может состоять в нескольких организациях одновременно
- Возможные статусы пользователя в рамках организации:

  - **Active**
      - Пользователь является участником организации и может выполнять действия согласно своим ролям/правам
      - Используется по умолчанию после успешного вступления (присоединения по ссылке)

  - **Deactivated**
      - Пользователь остаётся участником организации, но его доступ временно отключён
      - Роли и назначения сохраняются, но права не действуют, пока статус не вернётся в `Active`

  - **Removed**
      - Пользователь больше не является участником организации (доступ закрыт)
      - Используется при удалении участника из организации

### Приглашения
- Приглашения только по ссылке (invite-link)
- Возможные статусы приглашения по ссылке:

  - **Pending**
      - Приглашение активно, ожидает принятия
      - При принятии приглашения должны выполняться проверки: срок действия и текущий статус пользователя в организации

  - **Accepted**
      - Приглашение было принято конкретным пользователем
      - Повторное принятие не допускается

  - **Revoked**
      - Приглашение отозвано создателем/админом
      - Принять такое приглашение нельзя

  - **Expired**
      - Приглашение просрочено
